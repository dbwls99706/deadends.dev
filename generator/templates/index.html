deadends.dev

Structured failure knowledge for AI agents.

When AI agents hit an error, they search the web. But search results are written by humans, for humans — unstructured, environment-unaware, and often wrong for specific conditions.

deadends.dev tells you what NOT to try, what actually works (and under which conditions), and what error comes next if you take a particular path.

Paste your error message here to find matches instantly.

For AI Agents

Every page includes machine-readable data in multiple formats:

  • JSON API — GET /api/v1/{domain}/{slug}/{env}.json
  • Error index — GET /api/v1/index.json — all errors with regex patterns
  • OpenAPI spec — GET /api/v1/openapi.json
  • JSON-LD — Schema.org TechArticle + ErrorCanon in every page <head>
  • ai-summary — parseable <pre id="ai-summary"> block in every page
  • Stats — GET /api/v1/stats.json — dataset quality metrics
  • NDJSON — GET /api/v1/errors.ndjson — streaming format
  • llms.txt — GET /llms.txt — error listing for LLMs
  • Full dump — GET /llms-full.txt — complete database
